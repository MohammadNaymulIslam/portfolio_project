Linux System & AWS Instance Management

Project Goal

To deploy and manage cloud-based and on-premises Linux systems while implementing best practices in system hardening, remote access, and web server hosting.

Action & Technologies Used

  • Launched and managed EC2 instances on AWS, including key pair access and security group configuration.
  • Enabled and secured RDP/SSH access for remote management.
  • Configured Logical Volume Management (LVM) for scalable disk space management.
  • Applied Linux hardening techniques (disabling unused services, securing SSH, setting firewall rules).
  • Installed and configured Nginx to serve web content and reverse proxy apps.

Outcome

Gained hands-on experience in managing cloud and local Linux environments. Demonstrated ability to administer secure systems, automate deployments, and maintain high availability using AWS and Linux tools.
