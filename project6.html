Windows Server Domain & Active Directory

Project Goal

To design a centralized domain environment for user authentication, policy enforcement, and secure resource access using Windows Server technologies.

Action & Technologies Used

  • Installed and configured Windows Server as a Domain Controller.
  • Created and managed Active Directory objects: users, groups, and organizational units.
  • Enforced security settings through Group Policy Objects (GPOs).
  • Configured file sharing with NTFS and Share permissions based on security groups.
  • Tested login scripts, drive mappings, and GPO filtering for different departments.

Outcome

This project reinforced Windows Server administration and centralized identity management. Demonstrated skills in deploying secure, policy-driven IT environments that mirror enterprise standards.
